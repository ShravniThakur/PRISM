# Module 1 — Text-Based Threat Detection

**PRISM | Layer 2 | NLP Signal Extraction**

> Detects phishing emails, pump-and-dump social media posts, fraudulent voice call transcripts, and other text-based financial fraud targeting Indian retail investors using a fine-tuned **FinBERT** classifier.

## Quick Start


### 2. Generate & Prepare the Dataset

```bash
cd backend/layer2/module1/scripts

# Full run (generates 250 synthetic samples per persona + merges real data)
python prepare_text_dataset.py

# Customise sample count and seed
python prepare_text_dataset.py --samples-per-persona 500 --seed 99

# Re-use an existing synthetic_batch.json (skip generation)
python prepare_text_dataset.py --skip-synthetic
```

**Output:** `../data/processed/finbert_text_training_data.csv`

### 3. Train the FinBERT Baseline

```bash
python train_finbert_baseline.py

# GPU training with FP16
python train_finbert_baseline.py --fp16 --batch-size 32 --epochs 5

# CPU-only (slower)
python train_finbert_baseline.py --no-cuda --batch-size 8

# Custom paths
python train_finbert_baseline.py \
  --data-path  ../data/processed/finbert_text_training_data.csv \
  --output-dir ../models/finbert_baseline
```

**Output:** Fine-tuned model + tokeniser saved to `../models/finbert_baseline/`

### `train_finbert_baseline.py`

| Argument | Default | Description |
|---|---|---|
| `--data-path` | `../data/processed/finbert_text_training_data.csv` | Input CSV |
| `--output-dir` | `../models/finbert_baseline` | Model save path |
| `--epochs` | `3` | Training epochs |
| `--batch-size` | `16` | Per-device batch size |
| `--max-length` | `256` | Tokeniser max token length |
| `--lr` | `2e-5` | Learning rate |
| `--eval-split` | `0.15` | Validation fraction |
| `--test-split` | `0.10` | Test fraction |
| `--seed` | `42` | Random seed |
| `--fp16` | `False` | Mixed-precision (GPU only) |
| `--no-cuda` | `False` | Force CPU training |

**Training configuration:**

- **Base model:** `ProsusAI/finbert` (BERT trained on financial corpora)
- **Task:** Binary sequence classification (`SAFE=0`, `THREAT=1`)
- **Optimiser:** AdamW with cosine LR schedule
- **Warmup ratio:** 6 % of total steps
- **Weight decay:** 0.01
- **Early stopping:** patience = 2 epochs on `f1_macro`
- **Best checkpoint:** loaded automatically at end of training
- **Metrics:** Accuracy, F1 (macro), Precision, Recall

## Data Sources

### Raw — FinancialPhraseBank-v1.0
- **Source:** Malo et al. (2014). "Good Debt or Bad Debt: Detecting Semantic Orientations in Economic Texts". *JASIST*.
- **File:** `Sentences_AllAgree.txt` — 2,264 sentences annotated with unanimous agreement.
- **Usage:** Treated as **Label 0 (safe)** — legitimate financial news language.
- **Delimiter:** `@` (sentence@sentiment)

### Raw — PhishingEmailDataset / Ling.csv
- **Source:** Ling et al. spam/phishing corpus.
- **Columns:** `subject`, `body`, `label` (0=ham, 1=phishing)
- **Usage:** Label column mapped directly; combined subject+body as text.

### Raw — PhishingEmailDataset / Nigerian_Fraud.csv
- **Source:** Nigerian advance-fee fraud email dataset.
- **Usage:** All rows mapped to **Label 1 (threat)**.

### Synthetic — `synthetic_batch.json`
- **Generated by:** `prepare_text_dataset.py` using parameterised templates.
- **Format:** `[{"text": "...", "label": 0|1, "source_type": "email|social_media|voice_transcript"}]`
- **LLM integration:** Replace `_mock_llm_call()` in `prepare_text_dataset.py` with a real Gemini/Claude API call.

## Plugging in a Real LLM

In `prepare_text_dataset.py`, find `_mock_llm_call()` and replace its body:

```python
# --- Gemini (google-generativeai) ---
import google.generativeai as genai
genai.configure(api_key=os.environ["GEMINI_API_KEY"])
_genai_model = genai.GenerativeModel("gemini-1.5-flash")

async def _mock_llm_call(persona_key: str, slots: dict) -> str:
    prompt = (
        f"Generate one realistic {persona_key} example for PRISM fraud detection training. "
        f"Context variables: {slots}. Return only the text, no labels or JSON."
    )
    response = await _genai_model.generate_content_async(prompt)
    return response.text.strip()
```

```python
# --- Claude (anthropic) ---
import anthropic
_claude = anthropic.AsyncAnthropic(api_key=os.environ["ANTHROPIC_API_KEY"])

async def _mock_llm_call(persona_key: str, slots: dict) -> str:
    prompt = f"Generate one {persona_key} example. Variables: {slots}"
    msg = await _claude.messages.create(
        model="claude-3-5-haiku-20241022",
        max_tokens=512,
        messages=[{"role": "user", "content": prompt}]
    )
    return msg.content[0].text.strip()
```

The trained model is consumed by PRISM's Layer 2 scoring engine:

```
Incoming message / document
        ↓
  Module 1 (Text)   → threat_score_text   [0.0 – 1.0]
  Module 2 (Tabular)→ threat_score_tabular [0.0 – 1.0]
        ↓
  Ensemble Aggregator → final_risk_score
        ↓
  SEBI Alert Dashboard
```

## Notes & Known Limitations

- `ProsusAI/finbert` was originally pre-trained for 3-class sentiment (positive/negative/neutral). The classification head is replaced (`ignore_mismatched_sizes=True`) and fine-tuned from scratch for binary threat detection.
- `max_length=256` truncates long emails. Increase to 512 (memory permitting) for whole-email classification.
- Synthetic data quality improves significantly with a real LLM API — the mock templates are intentionally diverse but deterministic.
- For Windows: `dataloader_num_workers=0` is set by default to avoid multiprocessing issues. Linux/Mac users can increase this for faster data loading.
