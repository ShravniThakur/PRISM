# Module 1 — Text-Based Threat Detection

**PRISM | Layer 2 | NLP Signal Extraction**

> Detects phishing emails, pump-and-dump social media posts, fraudulent voice call transcripts, and other text-based financial fraud targeting Indian retail investors using a fine-tuned **FinBERT** classifier combined with a **URL typo-squat analyzer**.

## Components Overview

Module 1 has **two parallel analysis engines** that combine into a single `final_text_score`:

| Engine | File | Output |
|---|---|---|
| **FinBERT Classifier** | `inference_pipeline.py` | `model_confidence` (0.0–1.0) |
| **URL Typo-Squat Analyzer** | `url_analyzer.py` | `is_url_threat` (bool) + flagged URL list |
| **Score Fusion** | `inference_pipeline.py` | `final_text_score` (0.0–1.0) with penalty logic |

## Quick Start

### 1. Generate & Prepare the Dataset

```bash
cd backend/layer2/module1/scripts

# Full run (generates 250 synthetic samples per persona + merges real data)
python prepare_text_dataset.py

# Customise sample count and seed
python prepare_text_dataset.py --samples-per-persona 500 --seed 99

# Re-use an existing synthetic_batch.json (skip generation)
python prepare_text_dataset.py --skip-synthetic
```

**Output:** `../data/processed/finbert_text_training_data.csv`

### 2. Deduplicate the Dataset (Recommended before training)

```bash
python audit_and_dedup.py
```

**Output:** `../data/processed/finbert_text_training_data_deduped.csv` — 7,814 rows with a `split` column, zero exact/near-duplicates, zero train/test leakage.

### 3. Train the FinBERT Baseline

```bash
python train_finbert_baseline.py

# GPU training with FP16
python train_finbert_baseline.py --fp16 --batch-size 32 --epochs 5

# CPU-only (slower)
python train_finbert_baseline.py --no-cuda --batch-size 8

# Point at the deduped dataset (recommended)
python train_finbert_baseline.py \
  --data-path  ../data/processed/finbert_text_training_data_deduped.csv \
  --output-dir ../models/finbert_baseline
```

**Output:** Fine-tuned model + tokeniser saved to `../models/finbert_baseline/`

### 4. Run Inference

```bash
python inference_pipeline.py
```

Or programmatically:

```python
import asyncio
from layer2.module1.scripts.inference_pipeline import TextThreatAnalyzer

analyzer = TextThreatAnalyzer()
result = asyncio.run(
    analyzer.analyze_message(
        text="Dear Investor, your SEBI account is suspended. Verify OTP at https://sebii.gov.in/kyc",
        source_type="email"
    )
)
print(result["final_text_score"])   # e.g. 0.97
```

- **Task:** Binary sequence classification (`SAFE=0`, `THREAT=1`)

- **Warmup ratio:** 6% of total steps
- **Weight decay:** 0.01
- **Early stopping:** patience = 2 epochs on `f1_macro`
- **Best checkpoint:** loaded automatically at end of training
- **Metrics:** Accuracy, F1 (macro), Precision, Recall

### `inference_pipeline.py` (NEW)

The **main orchestrator** for all Module 1 analysis. Loaded once at server startup by `router.py` and called per-request.

**Full output contract** (always returned, even on error):

```json
{
    "final_text_score"     : 0.97,
    "model_confidence"     : 0.95,
    "url_analysis"         : {
        "urls_found"       : ["https://sebii.gov.in/kyc"],
        "suspicious_urls"  : [{"url": "...", "reason": "Typosquat of sebi.gov.in"}],
        "is_url_threat"    : true
    },
    "human_readable_reason": "High-confidence phishing attempt with a typosquatted SEBI domain.",
    "source_type"          : "email",
    "model_used"           : "finbert_baseline",
    "error"                : null
}
```

**Score fusion logic:**
1. Raw `model_confidence` from FinBERT is the base score.
2. If `is_url_threat = true`, a **URL penalty** is applied and the score is clamped to 1.0.
3. `final_text_score` is what gets returned to the Random Forest Scoring Engine.

**Threading:** `analyze_message()` is `async`. FinBERT inference runs in a `ThreadPoolExecutor` so it never blocks the FastAPI event loop.

### `url_analyzer.py` (NEW — hardened, 27/27 stress-test PASS)

A pure-Python, zero-dependency URL typo-squat heuristic that detects impersonation of Indian financial brands.

**Whitelist covered:** SEBI, NSE, BSE, RBI, IRDAI, Zerodha, Groww, Upstox, HDFC, ICICI, Kotak, Paytm Money, Axis Bank, and more.

**Detection techniques:**
- Edit-distance / `SequenceMatcher` ratio (threshold: 0.70)
- Brand-token prefix & substring check for hyphenated phishing domains (`sebi-secure-verify.co`)
- Homoglyph normalization (0->o, 5->S, 8->B, 1->l)
- Compound TLD parsing (`.gov.in`, `.co.in`, `.gov.org`)
- Brand-stem check (3-char prefix for 7+ char brands, e.g. `nse` -> `nseindia`)
- Legitimate subdomain allowlist guard (`api.zerodha.com` -> NOT flagged)

**Stress-test results (27 cases — 0 false positives, 0 false negatives):**

| Category | Cases | Result |
|---|---|---|
| Typosquats (char substitution) | 8 | All flagged |
| Homoglyph attacks | 4 | All flagged |
| Wrong TLD spoofs | 4 | All flagged |
| Legitimate subdomains (must NOT flag) | 4 | None flagged |
| Shortened URLs (documented blind spot) | 3 | Correctly not flagged |
| Unknown / edge-case domains | 3 | Not flagged |
| Mixed regression | 1 | Correctly flagged |

### `audit_and_dedup.py` (NEW)

Audits the original training CSV for data integrity issues and produces a clean deduplicated dataset.

**What it does:**
1. **Exact dedup:** MD5 hash on normalised text — removed 28 extra copies.
2. **Near-dedup:** `TfidfVectorizer(analyzer='char_wb', ngram_range=(3,5))` cosine similarity >= 0.85 — removed 838 secondary rows.
3. **Leakage check:** Detects rows that straddle the train/test split boundary — found and fixed 10 leaked rows.
4. **Fresh stratified split:** Clean 80/20 split (seed=42) on deduplicated data, stored in `split` column.

**Output:** `../data/processed/finbert_text_training_data_deduped.csv` (7,814 rows: 6,251 train / 1,563 test)

### `build_eval_holdout.py` & `add_uci_spam_holdout.py` (NEW)

## Data Sources

### Raw — FinancialPhraseBank-v1.0
- **Source:** Malo et al. (2014). "Good Debt or Bad Debt: Detecting Semantic Orientations in Economic Texts". *JASIST*.
- **File:** `Sentences_AllAgree.txt` — 2,264 sentences with unanimous agreement.
- **Usage:** Treated as **Label 0 (safe)** — legitimate financial news language.

### Raw — PhishingEmailDataset / Ling.csv
- **Source:** Ling et al. spam/phishing corpus.
- **Columns:** `subject`, `body`, `label` (0=ham, 1=phishing)
- **Usage:** Subject + body combined as text; label mapped directly.

### Raw — PhishingEmailDataset / Nigerian_Fraud.csv
- **Source:** Nigerian advance-fee fraud email dataset.
- **Usage:** All rows mapped to **Label 1 (threat)**.

### Synthetic — `synthetic_batch.json`
- **Generated by:** `prepare_text_dataset.py` using parameterised templates.
- **Format:** `[{"text": "...", "label": 0|1, "source_type": "email|social_media|voice_transcript"}]`
- **Note:** 779 of 800 synthetic records were found verbatim in training CSV (21 dropped during balancing). No synthetic rows appear in the real-world holdout.

## Plugging in a Real LLM

In `prepare_text_dataset.py`, find `_mock_llm_call()` and replace its body:

```python
# --- Gemini (google-generativeai) ---
import google.generativeai as genai
genai.configure(api_key=os.environ["GEMINI_API_KEY"])
_genai_model = genai.GenerativeModel("gemini-1.5-flash")

async def _mock_llm_call(persona_key: str, slots: dict) -> str:
    prompt = (
        f"Generate one realistic {persona_key} example for PRISM fraud detection training. "
        f"Context variables: {slots}. Return only the text, no labels or JSON."
    )
    response = await _genai_model.generate_content_async(prompt)
    return response.text.strip()
```

```python
# --- Claude (anthropic) ---
import anthropic
_claude = anthropic.AsyncAnthropic(api_key=os.environ["ANTHROPIC_API_KEY"])

async def _mock_llm_call(persona_key: str, slots: dict) -> str:
```

## Notes & Known Limitations

- `ProsusAI/finbert` was originally pre-trained for 3-class sentiment. The classification head is replaced (`ignore_mismatched_sizes=True`) and fine-tuned from scratch for binary threat detection.
- `max_length=256` truncates long emails. Increase to 512 (memory permitting) for full-email classification.
- **URL blind spots:** Shortened URLs (`bit.ly`, `t.me`) are not resolved — flagged as `needs_review` in production. Phishing via legitimate services (Google Forms, Firebase) is not detectable by the domain heuristic alone.
- For **Windows**: `dataloader_num_workers=0` is set by default to avoid multiprocessing issues. Linux/Mac users can increase this for faster data loading.
- `TextThreatAnalyzer` is **not thread-safe at construction time** — instantiate once at app startup and share the instance across requests.
